# 1안

프로젝트 주제(초안)  
문제 영역:  대출, 보험, 카드, 투자 등 금융 상품 약관 읽기의 복잡성을 덜기 위해 LLM 활용 금융 약관 분석 서비스  
주요 사용자(타깃):  
금융 지식이 부족한 일반 금융 소비자  
사회초년생, 청년층  
해결하고 싶은 Pain Point:  
금융 약관이 지나치게 길고 전문 용어가 많아 이해하기 어려움  
불리한 조항(위약금, 해지 조건, 수수료 등)을 사용자가 인지하지 못한 채 계약함  
약관을 읽는 데 시간이 많이 들고, 핵심만 파악하기 어려움  
서비스/기능 아이디어 (rough):  
금융 약관(PDF/문서)을 업로드하면 MCP 서버를 통해 분석  
LLM이 약관을 요약하고, 사용자에게 불리한 조항을 자동 하이라이팅  
자연어 질의 지원  
예) “중도 해지 시 불이익 있어?”, “수수료는 언제 발생해?”  
초보자/일반 사용자 기준으로 쉬운 언어로 설명 제공  
구현 가능성  
LLM: GPT API 활용 및 파이썬 라이브러리 활용해 구현 가능  
• • 단점: 파이썬으로 구현해본 경험은 많으나 자바로는 따로 서버를 띄웠어서 시스템 설계가 번거로워질 것 같습니다.

개요  
해결하고 싶은 Pain Point:  
사업자들은 사업을 진행하기 위해 입지 정보를 얻어야 하며, 다양한 수치적 통계 자료가 필요하다.  
  
  
  
  
서비스 기능/ 아이디어  
매장 혼잡도를 통해 유동인구 정보를 제공하는 B2B 서비스  
  
공공 API를 이용해 특정 지역의 실시간 혼잡도를 지도 위에 시각화한다.  
  
서비스를 제공하는 사업자들에게 중요한 통계 정보를 제공  
  
이 외에도 추가로 제공할 수 있는 차별점  
주변 건물 정보 등 사업자 입장에서 입지를 고려할 수 있는 정보들  
  
개발 범위?  
MVP 개발 기준  
- 특정 도시 1곳

기술 개발 관점 특징  
- 주기적 데이터 수집  
- 지도 시각화 (Heatmap 또는 Grid)  
  
etc
