# 트러블 슈팅

## Object Mapper 직렬화 이슈 (cfc5380d)
문제 상황 및 원인 : DB나 3자 API 등 외부와 연동되는 기능이 직렬화/역직렬화가 작동하지 않는 이슈

원인 : Redis Config에서 private로 설정되어 있던 ObjectMapper를 public 빈으로 바꿔버리면서 전역 설정이 변경, 직렬화 기준을 설정하지 않아 그 기준이 모호해짐

해결책 : @JsonProperty를 이용해 필드명을 명시하거나 package-private 필드를 private로 명시하여 해결.


## Redis Geospatial 직렬화 이슈 (5a3fe25)

문제 상황 : Redis에 충전소 위치를 저장하고 저장된 값을 가져오려 하던 중 Json을 Map 컬렉션으로 인식해 자바 객체로 변환하지 못하는 문제 발생

원인 : Redis의 ObjectMapper가 클래스 정보를 담지 않아 특정 클래스로의 직렬화 기준이 존재하지 않음.

해결책 : 해당 객체에 대한 전용 Serializer를 만들어 항상 해당 객체로 변환되도록 설정하여 해결

## 충전소 검색 기능 개선 과정
1. 기존 충전소 찾기 기능은 화면을 축소해서 검색 시 모든 충전소를 마커로 표기해 JS 처리량 한계

해결 : 카카오 맵 API의 클러스터링 기능을 이용해 전체 마커 수를 줄임

결과 : 여전히 프론트엔드에서 연산 부하가 가해지는 문제는 있으나 데이터 특성을 고려해 일반적으로 감당할 수 있다고 판단.

2. 백엔드에서 DB를 사용하기 때문에 API의 속도는 여전히 느림

해결 : 검색 속도를 개선하기 위해 Redis Geospatial 기능을 사용해 Redis 캐싱 및 위경도 기반 검색 구현, 서버 로드 시 캐시 웜업으로 초기 검색 속도를 높임.
