페이징 검색 기능 구현하다가 발생했다.
페이징할 때는 결과 검색이랑, 카운팅해주는 거 두 쿼리가 필요한데, 내가 두 번째 쿼리에 대해서 만들어주지 않았다가 오류가 발생했다.
정확한 오류에 대해서는 딥-다이브 하지 않아서 모르겠다. 이건 꼭 두 개다 작성안해도 될 것 같은데 Criteria API 쓸 떼 페이징 쿼리 어떻게 짜는 지 물어봐야 겠음. JPQL 직접 구현으로밖에 못하나? 그럼 Type-safe 포기해야 할까?

하면서 알게된 사실, pageable은 sort 옵션을 제공한다.
굳이 request에서 정렬 기능을 넣을 필요 없었던 것이다...

페이징으로 삽질하면서 생각난건데, pageable이 sort 옵션을 제공하는데, 사용자가 의도하지 않은 검색 결과를 볼 수도 있습니다. 또는 보안이 중요할 경우도 있는데, 이걸 노출시키지 않고 아예 새로운 필드를 사용하는 경우도 있는지 궁금함.
