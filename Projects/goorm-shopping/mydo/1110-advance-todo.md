기술 부채
~~- 기존 단일 쿠폰 사용 기능을 여러 쿠폰 사용하도록 확장 필요~~
- 지연 로딩 사용으로 인해 발급된 쿠폰에서 쿠폰 정책을 가져오면서 추가적인 DB I/O 발생 가능

기술 부채
- 현재 RequestDate 필드는 요청 시간을 서버의 현재 시간과 비교하는 유효성 검증으로 사용됨. 차후 RequestDate가 빠를 수록 우선순위를 높여주는 개선 필요

- 쿠폰 생성 후 발급 가능 시간 처리 방안
	- 현재 쿠폰은 수동으로 활성화 필드만을 이용해 발급 가능 여부를 처리하고 있음.
	- 시스템에서 보완해줄 필요성 존재, kafka를 통해 이벤트 처리 방안

- ~~최소 금액, 최대 할인액, 특정 상품 전용 할인 등의 더 복잡한 쿠폰 할인 정책 확장 고려~~
	- 전략 패턴으로 리팩토링
	- 여러 쿠폰 사용과 결합 시 NP 문제로 발전 가능성 존재
		- **이 문제의 경우, 단일 쿠폰 전략만 복잡한 정책 사용, 간단한 정액과 비율 할인만 다중 사용으로 전환함.**
